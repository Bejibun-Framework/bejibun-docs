`ace` is Bejibun's command-line companion.
It scaffolds code, runs migrations and seeders, processes queues, and toggles maintenance mode.

## Discovering commands

```bash
bun ace                  # list all commands
bun ace help <command>   # detailed help for one command
bun ace --version        # framework version
```

## Generators

Scaffold application components:

| Command                  | Creates                               |
| ------------------------ | ------------------------------------- |
| `make:command <name>`    | A command in `commands/`              |
| `make:controller <name>` | A controller in `app/controllers/`    |
| `make:job <name>`        | A job in `app/jobs/`                  |
| `make:middleware <name>` | A middleware in `app/middlewares/`    |
| `make:migration <name>`  | A migration in `database/migrations/` |
| `make:model <name>`      | A model in `app/models/`              |
| `make:seeder <name>`     | A seeder in `database/seeders/`       |
| `make:validator <name>`  | A validator in `app/validators/`      |

## Database

| Command            | Description                           |
| ------------------ | ------------------------------------- |
| `db:seed`          | Run seeders                           |
| `migrate:fresh`    | Drop all tables and re-run migrations |
| `migrate:latest`   | Run pending migrations                |
| `migrate:rollback` | Roll back the latest batch            |
| `migrate:status`   | Show migration status                 |

## Queues & scheduling

| Command         | Description                     |
| --------------- | ------------------------------- |
| `schedule:work` | Run the task scheduler          |
| `queue:flush`   | Flush all failed jobs           |
| `queue:retry`   | Retry a failed job              |
| `queue:work`    | Process queued jobs as a daemon |

## Maintenance & packages

| Command                 | Description                            |
| ----------------------- | -------------------------------------- |
| `install <packages...>` | Install and configure Bejibun packages |
| `maintenance:down`      | Put the app into maintenance mode      |
| `maintenance:up`        | Bring the app back online              |
| `package:configure`     | Configure a package after installation |
| `route:list`            | List routes / build the OpenAPI spec   |

## Writing your own command

Generate one with `make:command`, then fill in the signature and `handle` method:

```ts commands/SyncUsersCommand.ts
export default class SyncUsersCommand {
    /**
     * The name and signature of the console command.
     *
     * @var $signature string
     */
    protected $signature: string = "users:sync";

    /**
     * The console command description.
     *
     * @var $description string
     */
    protected $description: string = "Sync users from the upstream service";

    /**
     * The options or optional flag of the console command.
     *
     * @var $options Array<Array<string>>
     */
    protected $options: Array<Array<string>> = [];

    /**
     * The arguments of the console command.
     *
     * @var $arguments Array<Array<string>>
     */
    protected $arguments: Array<Array<string>> = [];

    public async handle(options: any, args: Array<string>): Promise<void> {
        // command logic
    }
}
```

Commands are auto-discovered from `commands/` -- once the file exists with a `$signature` and `handle()`,
it appears in `bun ace`.

<Tip>
    Schedule any command to run automatically by referencing its `$signature` in `commands/Kernel.ts`.
    See [Task Scheduling](/v0.4/guides/background/scheduler).
</Tip>
