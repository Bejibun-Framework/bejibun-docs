The scheduler runs `ace` commands on a recurring schedule using a fluent, Laravel-style API -- no need to edit system crontabs.

## Defining schedules

Schedules are declared in `commands/Kernel.ts`, inside the `schedule()` method:

```ts commands/Kernel.ts
import type Schedule from "@bejibun/core/facades/Schedule";

export default class Kernel {
    public schedule(schedule: Schedule): void {
        schedule.command("hello:world").everyMinute();
        schedule.command("reports:daily").dailyAt("02:30");
        schedule.command("cache:prune").everyTenMinutes();
    }
}
```

`schedule.command(signature)` targets an `ace` command by its `$signature`, then you chain a frequency method.

## Running the scheduler

```bash
bun ace schedule:work
```

Run this as a long-lived daemon (one per deployment) and it dispatches due commands at their scheduled times.

## Frequency methods

<AccordionGroup>
  <Accordion title="Seconds" icon="stopwatch">
    `everySecond`, `everyTwoSeconds`, `everyFiveSeconds`, `everyTenSeconds`, `everyFifteenSeconds`, `everyTwentySeconds`, `everyThirtySeconds`
  </Accordion>
  <Accordion title="Minutes" icon="clock">
    `everyMinute`, `everyTwoMinutes`, `everyThreeMinutes`, `everyFourMinutes`, `everyFiveMinutes`, `everyTenMinutes`, `everyFifteenMinutes`, `everyThirtyMinutes`
  </Accordion>
  <Accordion title="Hours" icon="hourglass">
    `hourly`, `hourlyAt(minute)`, `everyOddHour(minute?)`, `everyTwoHours(minute?)`, `everyThreeHours`, `everyFourHours`, `everySixHours`
  </Accordion>
  <Accordion title="Days, weeks, months, years" icon="calendar">
    `daily`, `dailyAt("HH:mm")`, `twiceDaily(h1, h2)`, `twiceDailyAt(h1, h2, minute)`, `weekly`, `weeklyOn(day, "HH:mm")`, `monthly`, `monthlyOn(day, "HH:mm")`, `twiceMonthly(d1, d2, "HH:mm")`, `yearly`, `yearlyOn(month, day, "HH:mm")`
  </Accordion>
</AccordionGroup>

## Timezone and raw cron

Set a timezone before the frequency, or supply a raw cron expression:

```ts
schedule.command("backup:db").weeklyOn(0, "03:00").timezone("Asia/Jakarta");

schedule.command("custom").cron("0 */15 * * * *"); // 6-field, seconds-precision
```

<Note>
Cron expressions are **6-field** (seconds first): `second minute hour day month weekday`.
</Note>
