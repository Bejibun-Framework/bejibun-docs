`@bejibun/redis` wraps Bun's native Redis client with a convenient facade.
It powers the Redis cache driver and is also available directly for keys, counters, pipelines, and pub/sub.

## Installation

```bash
bun ace install @bejibun/redis
```

This installs the package and drops `config/redis.ts`. Connection details come from the environment:

```env
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DATABASE=0
REDIS_MAX_RETRIES=10
REDIS_CONNECTION=local
```

## Keys and values

```ts
import Redis from "@bejibun/redis";

await Redis.set("session:1", token, 3600);          // value with TTL (seconds)
const value = await Redis.get("session:1");
await Redis.del("session:1");

const exists = await Redis.exists("session:1");
const keys = await Redis.keys("session:*");
```

## Counters

```ts
await Redis.incr("page:views");
await Redis.incrBy("page:views", 10);
await Redis.decr("stock:42");
await Redis.expire("page:views", 86400);
```

## Pipelines

Batch multiple commands in one round trip:

```ts
await Redis.pipeline((pipe) => {
    pipe.set("a", 1);
    pipe.set("b", 2);
});
```

## Pub/Sub

```ts
await Redis.subscribe("orders", (message) => {
    // handle incoming message
});

await Redis.publish("orders", order);
```

## Multiple connections

Target a named connection defined in `config/redis.ts`:

```ts
await Redis.connection("analytics").incr("events");
```

## API reference

| Method           | Purpose                                                                        |
| ---------------- | ------------------------------------------------------------------------------ |
| `setClient`      | Register a Redis client configuration, optionally with a connection name.      |
| `connection`     | Retrieve the registered Redis client instance by connection name.              |
| `connect`        | Establish a Redis connection.                                                  |
| `disconnect`     | Close an active Redis connection.                                              |
| `set`            | Store a value by key, with an optional TTL.                                    |
| `get`            | Retrieve the value of a key.                                                   |
| `del`            | Delete a key from Redis.                                                       |
| `exists`         | Check whether a key exists.                                                    |
| `keys`           | Return all keys matching a pattern.                                            |
| `incr`           | Increment a numeric value by 1.                                                |
| `decr`           | Decrement a numeric value by 1.                                                |
| `incrBy`         | Increment a numeric value by the specified amount.                             |
| `decrBy`         | Decrement a numeric value by the specified amount.                             |
| `ttl`            | Get the remaining time-to-live of a key.                                       |
| `expire`         | Set or update the expiration time of a key.                                    |
| `publish`        | Publish a message to a Pub/Sub channel.                                        |
| `subscribe`      | Subscribe to a Pub/Sub channel and receive messages.                           |
| `pipeline(fn)`   | Execute multiple Redis commands in a single pipeline for improved performance. |
| `on(event, fn)`  | Register an event listener for `connect`, `disconnect`, or `error` events.     |
| `off(event, fn)` | Remove a previously registered event listener.                                 |

<Tip>
    Most methods accept trailing `connection?` and `disconnectAfter?` arguments for one-off operations on a specific
    connection.
</Tip>
