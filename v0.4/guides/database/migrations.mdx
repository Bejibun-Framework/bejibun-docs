Migrations are version control for your database schema. Bejibun uses [Knex.js](https://knexjs.org) migrations,
managed through the `ace` CLI.

## Configuration

The database connection lives in `config/database.ts`. PostgreSQL is the default client:

```ts config/database.ts
import type {Knex} from "knex";

const config: Knex.Config = {
    client: "pg",
    connection: {
        host: env("DB_HOST"),
        port: env("DB_PORT"),
        user: env("DB_USER"),
        password: env("DB_PASSWORD"),
        database: env("DB_DATABASE")
    },
    migrations: {
        extension: "ts",
        directory: "./database/migrations",
        schemaName: "public",
        tableName: "migrations"
    },
    pool: {
        min: 0,
        max: 10
    },
    seeds: {
        extension: "ts",
        directory: "./database/seeders"
    }
};

export default config;
```

<Note>
    The database layer is provided by `@bejibun/database`.
    Install it with `bun ace install @bejibun/database` if it isn't already present.
</Note>

## Creating a migration

```bash
bun ace make:migration create_users_table
```

This creates a timestamped file in `database/migrations/` with `up` and `down` functions:

```ts database/migrations/20260101_000001_create_users_table.ts
import type {Knex} from "knex";
import UserModel from "@/app/models/UserModel";

export function up(knex: Knex): Knex.SchemaBuilder {
    return knex.schema.createTable(UserModel.tableName, (table: Knex.TableBuilder) => {
        table.bigIncrements("id");
        table.string("name");
        table.string("email").unique();
        table.timestamps(true, true);
        table.timestamp("deleted_at");
    });
}

export function down(knex: Knex): Knex.SchemaBuilder {
    return knex.schema.dropTable(UserModel.tableName);
}
```

## Running migrations

<CodeGroup>
    ```bash Apply pending
    bun ace migrate:latest
    ```

    ```bash Rollback latest batch
    bun ace migrate:rollback
    ```

    ```bash Drop all + re-run
    bun ace migrate:fresh
    ```

    ```bash Show status
    bun ace migrate:status
    ```
</CodeGroup>

Example output:

```text
✔ Batch 1 finished
✔ 20260101_000001_create_users_table.ts
```

<Warning>
    `migrate:fresh` and `migrate:rollback` prompt for confirmation because they're destructive.
    `migrate:fresh` drops **all** tables before re-running every migration -- never run it against production data.
</Warning>

## Column types

Use the full Knex schema builder inside `up` --
`bigIncrements`, `string`, `integer`, `boolean`, `timestamp`, `json`, `index`, `foreign`, and more.
See the [Knex schema docs](https://knexjs.org/guide/schema-builder.html) for the complete API.
