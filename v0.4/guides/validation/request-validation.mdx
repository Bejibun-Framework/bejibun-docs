Once you've defined a [validator](/v0.4/guides/validation/creating-validators),
run it inside a controller with `super.validate()`.

## Validating a request

Parse the request, validate it, then proceed:

```ts app/controllers/UserController.ts
import BaseController from "@bejibun/core/bases/BaseController";
import UserModel from "@/app/models/UserModel";
import UserValidator from "@/app/validators/UserValidator";

export default class UserController extends BaseController {
    public async store(request: Bun.BunRequest): Promise<Response> {
        const body = await super.parse(request);
        await super.validate(UserValidator.store, body);

        const user = await UserModel.create({
            name: body.name as string
        });

        return super.response.setData(user).send();
    }
}
```

<Note>
    `validate()` returns the validated data and throws a `ValidatorException` on failure --
    there's no need to check a result flag.
    Execution stops and the [exception handler](/v0.4/guides/core-concepts/error-handling) formats the error response.
</Note>

## Validating route parameters

Route parameters land in the same parsed object, so they validate like any other field.
This pairs perfectly with the `exists` rule:

```ts app/validators/UserValidator.ts
public static get show(): ValidatorType {
    return super.validator.create({
        id: super.validator.number().min(1).exists(UserModel, "id")
    });
}
```

```ts app/controllers/UserController.ts
public async show(request: Bun.BunRequest): Promise<Response> {
    const body = await super.parse(request);
    await super.validate(UserValidator.show, body);

    const user = await UserModel.findOrFail(body.id as number | string);

    return super.response.setData(user).send();
}
```

## Typical CRUD validators

Each controller action references its matching schema:

| Action    | Validator          | Typical rules           |
| --------- | ------------------ | ----------------------- |
| `store`   | `Validator.store`  | required fields         |
| `show`    | `Validator.show`   | `id` + `exists`         |
| `update`  | `Validator.edit`   | `id` + `exists`, fields |
| `destroy` | `Validator.delete` | `id` + `exists`         |

<Tip>
    Validate **before** touching the database. Combined with `exists`, you get a clean validation error instead of a
    deeper failure when a referenced record is missing.
</Tip>
