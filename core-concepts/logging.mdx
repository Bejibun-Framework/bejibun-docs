# Logging

Logging is one of the most important aspects of application development and operations.

When applications run in production, logs provide visibility into what is happening inside the system. They help developers diagnose issues, monitor application health, audit behavior, and understand how users interact with the application.

A well-designed logging strategy can significantly reduce debugging time and improve system reliability.

---

# What Is Logging?

Logging is the process of recording events that occur during application execution.

Examples include:

- Incoming requests
- Authentication attempts
- Validation failures
- Database errors
- Background job execution
- External API calls
- System warnings

Example log entry:

```text
[2026-06-07 10:15:42] INFO User authenticated successfully
```

Logs create a historical record of application activity.

---

# Why Logging Matters

Without logs:

```text
Application Error

      ▼

```

With logs:


Benefits include:

# Writing Logs

The Logger service provides methods for recording events.

Example:

```ts
Logger.info(
  "User logged in"
);
```

Result:

```text
[INFO] User logged in
```

This is the most common form of logging.

---

# Log Levels

Not all log messages have the same importance.

Bejibun supports multiple log levels to categorize events.

---

## Emergency

The system is unusable.

Example:

```ts
Logger.emergency(
  "Database cluster unavailable"
);
```

Use only for critical failures.

---

## Alert

Immediate action is required.

Example:

```ts
Logger.alert(
  "Payment service unreachable"
);
```

---

## Critical

A critical component has failed.

Example:

```ts
Logger.critical(
  "Unable to connect to database"
);
```

---

## Error

An operation failed.

Example:

```ts
Logger.error(
  "User creation failed"
);
```

Errors typically require investigation.

---

## Warning

Unexpected behavior occurred but execution continues.

Example:

```ts
Logger.warning(
  "Rate limit threshold reached"
);
```

Warnings indicate potential issues.

---

## Notice

Important but expected events.

Example:

```ts
Logger.notice(
  "User password changed"
);
```

---

## Info

General application activity.

Example:

```ts
Logger.info(
  "User authenticated"
);
```

Most operational logs belong here.

---

## Debug

Detailed diagnostic information.

Example:

```ts
Logger.debug({
  query,
  executionTime
});
```

Debug logs are useful during development and troubleshooting.

---

# Choosing Log Levels

A simple guideline:

| Level | Usage |
|---------|---------|
| Emergency | Application unavailable |
| Alert | Immediate action required |
| Critical | Major component failure |
| Error | Failed operations |
| Warning | Unexpected situations |
| Notice | Significant events |
| Info | Normal activity |
| Debug | Diagnostic information |

Selecting appropriate levels improves observability.

# Structured Logging

Avoid:

```ts
Logger.info(
  "User 123 logged in"
);
```

Prefer:

```ts
Logger.info(
  "User authenticated",
  {
    userId: 123
  }
);
```

Structured logs provide machine-readable metadata.

Benefits include:

- Better searching
- Easier filtering
- Improved analytics
- Enhanced monitoring

---

# Contextual Logging

Logs become more valuable when context is included.

Example:

```ts
Logger.error(
  "Payment failed",
  {
    userId: 123,
    orderId: 456,
    provider: "stripe"
  }
);
```

Result:

```json
{
  "level": "error",
  "message": "Payment failed",
  "userId": 123,
  "orderId": 456,
  "provider": "stripe"
}
```

Context helps identify issues quickly.

# Log Channels

A channel defines where logs are written.

Example configuration:

```ts
export default {
  default: "file",

  channels: {
    file: {},
    console: {}
  }
};
```

Possible channels:

```text
File
Console
Remote Service
Database
Custom Driver
```

Applications may use multiple channels simultaneously.

---

# File Logging

File logging is the most common approach.

Example:

```text
storage/logs/
```

Files may look like:

```text
storage/logs/app.log
```

Example entry:

```text
[INFO] User created successfully
```

File logs are simple and reliable.

---

# Console Logging

Useful during development and containerized deployments.

Example:

```ts
Logger.info(
  "Server started"
);
```

Output:

```text
[INFO] Server started
```

Container platforms often collect console output automatically.

---

# Logging Exceptions

Exceptions should be logged with sufficient detail.

Example:

```ts
try {
  await processPayment();
} catch (error) {
  Logger.error(
    error.message
  );
}
```

Better:

```ts
Logger.error(
  "Payment processing failed",
  {
    error
  }
);
```

This preserves useful debugging information.

---

# Request Logging

Applications often log incoming requests.

Example:

```text
GET /users
POST /orders
DELETE /posts/1
```

Request logs help:

- Audit activity
- Measure traffic
- Diagnose issues
- Investigate incidents

---

## Example Middleware

```ts
export default class LoggingMiddleware {
  async handle(ctx, next) {
    Logger.info(
      "Incoming request",
      {
        method:
          ctx.request.method(),
        url:
          ctx.request.url()
      }
    );

    await next();
  }
}
```

# Performance Logging

Performance metrics can be logged for analysis.

Example:

```ts
const start =
  performance.now();

await service.execute();

Logger.debug(
  "Execution completed",
  {
    duration:
      performance.now() -
      start
  }
);
```

Useful for identifying bottlenecks.

---

# Security Logging

Security-sensitive events should always be logged.

Examples:

```text
Failed Login Attempt

Permission Denied

Password Reset

API Key Revoked
```

Example:

```ts
Logger.notice(
  "Failed login",
```

# Background Job Logging

Jobs should log important lifecycle events.

Example:

```ts

  "Email job started"
);
```

Later:

```ts
Logger.info(
  "Email job completed"
```

# Log Rotation

Logs grow continuously.

Without rotation:

```text
app.log
```

can become extremely large.

Rotation creates:

```text
app.log
app-2026-06-06.log
app-2026-06-07.log
```

Benefits:

- Reduced disk usage
- Easier maintenance
- Improved performance

Production environments should always implement log rotation.

# Logging Best Practices

### Log Important Events

Good:

```ts
Logger.info(
  "Order completed"
);
```

Avoid logging trivial operations excessively.

---

### Include Context

Good:

```ts
Logger.info(
  "Order completed",
  {
    orderId
  }
);
```

Context improves usefulness.

---

### Use Appropriate Levels

Do not log everything as:

```ts
Logger.error(...)
```

Use levels consistently.

---

### Avoid Sensitive Data

Never log:

```text
Passwords
Tokens
Secret Keys
Credit Card Numbers
```

Logs should not expose confidential information.

---

### Prefer Structured Logs

Structured data is easier to process and analyze.

---

# Common Mistakes

### Logging Secrets

Avoid:

```ts
Logger.info(password);
```

Sensitive information should never appear in logs.

---

### Logging Excessively

Too many logs create noise and increase storage costs.

---

### Ignoring Errors

Critical failures should always be logged.

---

### Using Debug Logs in Production

Verbose debugging may impact performance and overwhelm log storage.
