Seeders fill your database with data — defaults, reference tables, or sample records for development. They live in `database/seeders/` and run through the `ace` CLI.

## Creating a seeder

```bash
bun ace make:seeder tests_seeder
```

This creates a timestamped file exporting a `seed` function:

```ts database/seeders/20250929_000001_seeder_test.ts
import type {Knex} from "knex";
import TestModel from "@/app/models/TestModel";

export async function seed(knex: Knex): Promise<void> {
    for (const name of ["Name 1", "Name 2", "Name 3"]) {
        await TestModel.query(knex).insert({ name });
    }
}
```

<Note>
Pass the `knex` instance to `Model.query(knex)` so inserts run on the seeder's connection.
</Note>

## Running seeders

```bash
bun ace db:seed
```

Example output:

```text
✔ Seeding finished
✔ 20250929_000001_seeder_test.ts
```

## Patterns

- **Reference data** — seed lookup tables (roles, categories) that your app expects to exist.
- **Local development** — generate realistic sample rows so you can build against populated data.
- **Idempotency** — guard inserts (check existence first) if you intend to re-run a seeder safely.

```ts
export async function seed(knex: Knex): Promise<void> {
    const existing = await TestModel.query(knex).where("name", "Admin").first();
    if (!existing) {
        await TestModel.query(knex).insert({ name: "Admin" });
    }
}
```

<Tip>
Combine `migrate:fresh` and `db:seed` for a clean development reset: rebuild the schema, then repopulate it.
</Tip>
