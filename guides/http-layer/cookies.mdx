Bejibun does not ship a dedicated cookie helper. Because controllers work with the native `Bun.BunRequest` and
return standard `Response` objects, you read and write cookies using the platform's Web APIs.

<Note>
    A first-class cookie API is not yet available. The patterns below use native Bun/Web APIs that are fully supported
    today.
</Note>

## Reading cookies

Cookies arrive in the `Cookie` request header. Parse them from the raw request:

```ts app/controllers/SessionController.ts
import BaseController from "@bejibun/core/bases/BaseController";

export default class SessionController extends BaseController {
    public async show(request: Bun.BunRequest): Promise<Response> {
        const header = request.headers.get("cookie") ?? "";

        const cookies = Object.fromEntries(
            header.split(";").map((pair) => {
                const [key, ...rest] = pair.trim().split("=");
                return [key, decodeURIComponent(rest.join("="))];
            }).filter(([key]) => key)
        );

        return super.response.setData({ session: cookies.session_id }).send();
    }
}
```

Bun also exposes `request.cookies` (a `CookieMap`) on `Bun.BunRequest`:

```ts
const sessionId = request.cookies.get("session_id");
```

## Writing cookies

Cookies are set through the `Set-Cookie` response header. The `Response` facade builds the JSON body but
does not expose header control, so when you need to set a cookie, return a raw `Response`:

```ts
public async login(request: Bun.BunRequest): Promise<Response> {
    const body = await super.parse(request);

    return Response.json(
        { data: null, message: "Logged in", status: 200 },
        {
            status: 200,
            headers: {
                "Set-Cookie": `session_id=${crypto.randomUUID()}; HttpOnly; Path=/; SameSite=Lax; Max-Age=3600`
            }
        }
    );
}
```

<Warning>
    Always set security attributes on session cookies: `HttpOnly` (block JS access), `Secure` (HTTPS only, in
    production), and an appropriate `SameSite` value to mitigate CSRF.
</Warning>

## Persisting CORS headers

If you return a raw `Response` (instead of `super.response.send()`), remember that CORS headers are no longer
applied automatically. Merge them in manually:

```ts
import Cors from "@bejibun/cors";

return Response.json(payload, {
    headers: {
        ...Cors.init,
        "Set-Cookie": "session_id=...; HttpOnly; Path=/"
    }
});
```
