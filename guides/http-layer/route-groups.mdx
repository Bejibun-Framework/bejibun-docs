Route groups let you apply shared attributes — a URL **prefix**, **middleware**, or a controller **namespace** — to many routes at once. Bejibun also generates a full set of RESTful routes from a single `resource` call.

## Grouping routes

Chain modifiers on the `Router` facade, then call `.group([...])` with your routes:

```ts routes/api/test.ts
import Router from "@bejibun/core/facades/Router";
import TestMiddleware from "@/app/middlewares/TestMiddleware";
import LoggerMiddleware from "@/app/middlewares/LoggerMiddleware";

export default Router.prefix("test")
    .middleware(new TestMiddleware(), new LoggerMiddleware())
    .group([
        Router.get("get", "TestController@get"),
        Router.get("detail/:id", "TestController@detail"),
        Router.post("add", "TestController@add"),
        Router.delete("delete/:id", "TestController@delete")
    ]);
```

### Available modifiers

| Modifier | Purpose |
| --- | --- |
| `Router.prefix(path)` | Prepend a path segment to every route in the group |
| `Router.middleware(...m)` | Attach middleware instances to the group |
| `Router.namespace(dir)` | Override the controller resolution directory (default `app/controllers`) |
| `Router.x402(config?)` | Gate the group behind an [x402 payment](/) (requires `@bejibun/x402`) |

<Note>
Middleware runs in the order listed: the **first** middleware is the outermost wrapper and executes first on the way in. See [Middleware](/guides/http-layer/middleware).
</Note>

## Nesting groups

`routes/api.ts` is itself a group. Compose feature files under a shared prefix:

```ts routes/api.ts
import Router from "@bejibun/core/facades/Router";
import hello from "@/routes/api/hello";
import test from "@/routes/api/test";

// Everything is served under /api
export default Router.prefix("api").group([
    hello,
    test
]);
```

## Resource routes

`Router.resource()` generates the five conventional RESTful routes from a controller in one line:

```ts
import Router from "@bejibun/core/facades/Router";
import YourController from "@/app/controllers/YourController";

Router.resource("path", YourController);
```

This registers:

| Verb | URI | Controller action |
| --- | --- | --- |
| `GET` | `/path` | `index` |
| `POST` | `/path` | `store` |
| `GET` | `/path/:id` | `show` |
| `PUT` | `/path/:id` | `update` |
| `DELETE` | `/path/:id` | `destroy` |

Only actions that actually exist as methods on the controller are registered.

### Limiting resource routes

Use `only` or `except` to register a subset:

```ts
// register only index + store
Router.resource("path", YourController, {
    only: ["index", "store"]
});

// register everything except index + store
Router.resource("path", YourController, {
    except: ["index", "store"]
});
```

Valid actions: `"index"`, `"store"`, `"show"`, `"update"`, `"destroy"`.

<Tip>
`bun ace make:controller YourController` scaffolds a controller with all five resource actions ready to fill in.
</Tip>
