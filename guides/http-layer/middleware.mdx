Middleware wraps your route handlers, letting you run logic before (and after) a request reaches its controller -- authentication, logging, rate limiting, and more.

## The middleware contract

A middleware is a class with a `handle` method that receives the next handler and returns a new one:

```ts app/middlewares/TestMiddleware.ts
import type {HandlerType} from "@bejibun/core/types";
import Logger from "@bejibun/logger";

export default class TestMiddleware {
    public handle(handler: HandlerType): HandlerType {
        return async (request: Bun.BunRequest) => {
            Logger.setContext("TestMiddleware").debug(request.url);

            // call the next handler in the chain
            return handler(request);
        };
    }
}
```

Generate the boilerplate with:

```bash
bun ace make:middleware TestMiddleware
```

## Registering middleware

Attach middleware instances to a route group with `Router.middleware()`:

```ts routes/api/test.ts
import Router from "@bejibun/core/facades/Router";
import TestMiddleware from "@/app/middlewares/TestMiddleware";
import LoggerMiddleware from "@/app/middlewares/LoggerMiddleware";

export default Router.prefix("test")
    .middleware(new TestMiddleware(), new LoggerMiddleware())
    .group([
        Router.get("get", "TestController@get")
    ]);
```

<Note>
Middleware is applied so that the **first** middleware listed is the **outermost** wrapper -- it runs first on the way in and last on the way out.
</Note>

## Built-in middleware

Bejibun ships middleware that the server applies globally, toggled in `config/performance.ts`:

```ts config/performance.ts
const config = {
    middlewares: {
        limiter: true,
        maintenance: true
    }
};

export default config;
```

<AccordionGroup>
  <Accordion title="Rate limiter" icon="gauge-high">
    Limits requests per client IP. Configure the threshold in `config/limiter.ts`:

    ```ts config/limiter.ts
    const config = {
        limit: 30,
        duration: 60 // seconds
    };

    export default config;
    ```

    Exceeding the limit throws a `RateLimiterException`, returned as an error response.
  </Accordion>
  <Accordion title="Maintenance mode" icon="screwdriver-wrench">
    When the app is in maintenance mode, every request returns the maintenance message instead of routing. Toggle it with the CLI:

    ```bash
    bun ace maintenance:down   # enter maintenance mode
    bun ace maintenance:up     # return to live
    ```
  </Accordion>
</AccordionGroup>

## Rate limiting manually

You can also rate-limit specific actions with the `RateLimiter` facade:

```ts
import RateLimiter from "@bejibun/core/facades/RateLimiter";

await RateLimiter.attempt("send-otp:user-1", 5, async () => {
    // runs only if under the limit (5 attempts)
});

const blocked = await RateLimiter.tooManyAttempts("send-otp:user-1", 5);

await RateLimiter.clear("send-otp:user-1");
```

When omitted, `limit` and `duration` default to `60` / `60` seconds.
