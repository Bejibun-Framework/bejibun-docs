Validators are classes that extend `BaseValidator`. Each schema is a static getter, so you can reference it directly (for example `TestValidator.add`) without instantiating the class.

## Generating a validator

```bash
bun ace make:validator TestValidator
```

This creates `app/validators/TestValidator.ts` with resource-style schemas.

## Defining schemas

`super.validator` is the Vine.js root. There are two equivalent ways to define a schema.

<CodeGroup>
```ts create()
import type {ValidatorType} from "@bejibun/core/types/ValidatorType";
import BaseValidator from "@bejibun/core/bases/BaseValidator";

export default class TestValidator extends BaseValidator {
    public static get add(): ValidatorType {
        return super.validator.create({
            name: super.validator.string()
        });
    }
}
```

```ts compile(object())
import type {ValidatorType} from "@bejibun/core/types/ValidatorType";
import BaseValidator from "@bejibun/core/bases/BaseValidator";

export default class TestValidator extends BaseValidator {
    public static get store(): ValidatorType {
        return super.validator.compile(
            super.validator.object({
                name: super.validator.string()
            })
        );
    }
}
```
</CodeGroup>

## A full validator

Group every action's schema in one class:

```ts app/validators/TestValidator.ts
import type {ValidatorType} from "@bejibun/core/types/ValidatorType";
import BaseValidator from "@bejibun/core/bases/BaseValidator";
import TestModel from "@/app/models/TestModel";

export default class TestValidator extends BaseValidator {
    public static get detail(): ValidatorType {
        return super.validator.create({
            id: super.validator.number().min(1).exists(TestModel, "id")
        });
    }

    public static get add(): ValidatorType {
        return super.validator.create({
            name: super.validator.string()
        });
    }

    public static get edit(): ValidatorType {
        return super.validator.create({
            id: super.validator.number().min(1).exists(TestModel, "id"),
            name: super.validator.string()
        });
    }

    public static get delete(): ValidatorType {
        return super.validator.create({
            id: super.validator.number().min(1).exists(TestModel, "id")
        });
    }
}
```

<Tip>
Optional, nullable fields are expressed with Vine modifiers:

```ts
search: super.validator.string().nullable().optional()
```
</Tip>
