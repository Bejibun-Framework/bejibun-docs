---
title: "Soft Deletes"
description: "Keep records recoverable with deleted_at-based soft deletion."
icon: "trash-can-arrow-up"
---

Bejibun models support **soft deletes** out of the box. Instead of removing a row, `delete()` stamps the `deleted_at` column, and soft-deleted rows are automatically excluded from normal queries.

## Requirements

Your table needs a nullable `deleted_at` column and the model needs the `deleted_at` field:

```ts
// migration
table.timestamp("deleted_at");
```

```ts
// model
declare deleted_at: NullableTimestamp;
```

(The default `make:model` and `make:migration` stubs include these.)

## Soft deleting

```ts
await TestModel.find(id).delete();   // sets deleted_at = now()
```

The row stays in the database but disappears from default queries like `TestModel.all()`.

## Restoring

Clear `deleted_at` to bring a record back:

```ts
await TestModel.find(id).restore();
```

## Permanent deletion

To remove a row for good, bypass soft deletes with `forceDelete()`:

```ts
await TestModel.find(id).forceDelete();
```

## Querying trashed records

By default queries hide soft-deleted rows. Include or isolate them with:

<CodeGroup>
```ts With trashed
const all = await TestModel.withTrashed();    // active + trashed
```

```ts Only trashed
const trashed = await TestModel.onlyTrashed(); // trashed only
```
</CodeGroup>

## Method reference

| Method | Effect |
| --- | --- |
| `.delete()` / `.del()` | Soft delete (set `deleted_at`) |
| `.forceDelete()` | Permanent delete |
| `.restore()` | Clear `deleted_at` on trashed rows |
| `.withTrashed()` | Include trashed rows |
| `.onlyTrashed()` | Only trashed rows |

## Validating against trashed rows

The `exists` validation rule can opt into trashed rows — useful for a restore endpoint that must accept an already-deleted ID:

```ts
id: super.validator.number().min(1).exists(TestModel, "id", true)
```

```ts app/controllers/TestController.ts
public async restore(request: Bun.BunRequest): Promise<Response> {
    const body = await super.parse(request);
    await super.validate(TestValidator.restore, body);

    const test = await TestModel.find(body.id as number | string).restore();

    return super.response.setData(test).send();
}
```
