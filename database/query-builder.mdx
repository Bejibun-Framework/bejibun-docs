# Query Builder

The Query Builder provides a fluent interface for constructing and executing database queries.

Instead of writing raw SQL directly, developers can build queries using expressive TypeScript methods that are easier to read, maintain, and compose.

The Query Builder sits between your application and the database, allowing you to retrieve and manipulate data without sacrificing flexibility.

---

# What Is A Query Builder?

A Query Builder is an abstraction that converts method chains into database queries.

Example:

```ts
await User.query()
  .where("active", true)
  .orderBy("name");
```

Internally:

```sql
SELECT *
FROM users
WHERE active = true
ORDER BY name ASC;
```

The framework generates the SQL automatically.

---

# Why Use A Query Builder?

Without a Query Builder:

```ts
await db.raw(`
  SELECT *
  FROM users
  WHERE active = true
`);
```

As applications grow:

```text
Large SQL Strings

Repeated Queries

Manual Escaping

Reduced Readability
```

A Query Builder provides:

```text
Readability

Maintainability

Composability

Security

Database Portability
```

```text
Query Builder

      │
      ▼

```

The Query Builder handles SQL generation automatically.

# Starting A Query

Queries typically begin from a model.

Example:

```ts
User.query();
```

Flow:

```text
Model
   │
   ▼
Query Builder
```

This creates a new query instance targeting the model's table.

---

# Retrieving Records

Retrieve all records:

```ts
const users =
  await User.query();
```

Equivalent SQL:

```sql
SELECT *
FROM users;
```

Results are returned as model instances.

---

# Retrieving The First Record

Example:

```ts
const user =
  await User.query()
    .first();
```

SQL:

```sql
SELECT *
FROM users
LIMIT 1;
```

Useful when only a single result is needed.

---

# Filtering Results

Filtering restricts which records are returned.

Example:

```ts
await User.query()
  .where(
    "active",
    true
  );
```

SQL:

```sql
SELECT *
FROM users
WHERE active = true;
```

Filtering is one of the most common query operations.

---

# Multiple Conditions

Example:

```ts
await User.query()
  .where("active", true)
  .where("verified", true);
```

SQL:

```sql
SELECT *
FROM users
WHERE active = true
AND verified = true;
```

Conditions can be chained together.

---

# OR Conditions

Example:

```ts
await User.query()
  .where("role", "admin")
  .orWhere("role", "owner");
```

SQL:

```sql
SELECT *
FROM users
WHERE role = 'admin'
OR role = 'owner';
```

OR conditions provide additional flexibility.

---

# Comparison Operators

Examples:

```ts
.where("age", ">", 18)

.where("score", ">=", 100)

.where("price", "<", 50)
```

Generated SQL uses the specified comparison operator.

---

# IN Queries

Example:

```ts
await User.query()
  .whereIn(
    "id",
    [1, 2, 3]
  );
```

SQL:

```sql
SELECT *
FROM users
WHERE id IN (1, 2, 3);
```

Useful for filtering multiple values.

---

# NOT IN Queries

Example:

```ts
await User.query()
  .whereNotIn(
    "status",
    ["banned"]
  );
```

SQL:

```sql
SELECT *
FROM users
WHERE status
NOT IN ('banned');
```

---

# NULL Checks

Example:

```ts
await User.query()
  .whereNull(
    "deleted_at"
  );
```

SQL:

```sql
SELECT *
FROM users
WHERE deleted_at IS NULL;
```

Useful when working with optional values.

---

# NOT NULL Checks

Example:

```ts
await User.query()
  .whereNotNull(
    "email_verified_at"
  );
```

SQL:

```sql
SELECT *
FROM users
WHERE email_verified_at
IS NOT NULL;
```

---

# Ordering Results

Sort results by a column.

Example:

```ts
await User.query()
  .orderBy("name");
```

SQL:

```sql
ORDER BY name ASC;
```

Ordering controls result presentation.

---

# Descending Order

Example:

```ts
await User.query()
  .orderByDesc(
    "created_at"
  );
```

SQL:

```sql
ORDER BY created_at DESC;
```

Useful for showing newest records first.

---

# Limiting Results

Restrict the number of returned records.

Example:

```ts
await User.query()
  .limit(10);
```

SQL:

```sql
LIMIT 10;
```

Limits improve performance and reduce payload size.

---

# Offsetting Results

Example:

```ts
await User.query()
  .offset(20);
```

SQL:

```sql
OFFSET 20;
```

Commonly used for pagination.

---

# Pagination

Example:

```ts
await User.query()
  .paginate(1, 15);
```

Flow:

```text
Query
   │
   ▼
Paginator
   │
   ▼
Results + Metadata
```

Pagination helps manage large datasets efficiently.

---

# Selecting Columns

Retrieve only specific fields.

Example:

```ts
await User.query()
  .select(
    "id",
    "name"
  );
```

SQL:

```sql
SELECT id, name
FROM users;
```

This reduces unnecessary data transfer.

---

# Excluding Columns

Sometimes you only need a subset of fields.

Benefits:

```text
Smaller Responses

Better Performance

Improved Security
```

Always retrieve only what is necessary.

---

# Counting Records

Example:

```ts
await User.query()
  .count();
```

SQL:

```sql
SELECT COUNT(*)
FROM users;
```

Useful for dashboards and analytics.

---

# Existence Checks

Example:

```ts
await User.query()
  .where("email", email)
  .exists();
```

Result:

```ts
true
```

This avoids loading unnecessary data.

# Aggregate Functions

The Query Builder supports common aggregate operations.

Examples:

```ts
.count()

.sum()

.avg()

.min()

.max()
```

These operations execute directly in the database.

---

# Sum

Example:

```ts
await Order.query()
  .sum("total");
```

SQL:

```sql
SELECT SUM(total)
FROM orders;
```

---

# Average

Example:

```ts
await Product.query()
  .avg("price");
```

SQL:

```sql
SELECT AVG(price)
FROM products;
```

---

# Minimum And Maximum

Example:

```ts
await Product.query()
  .min("price");

await Product.query()
  .max("price");
```

Useful for reporting and statistics.

---

# Grouping Results

Example:

```ts
await Order.query()
  .groupBy("status");
```

SQL:

```sql
GROUP BY status;
```

Grouping enables aggregate reporting.

---

# HAVING Clauses

Example:

```ts
await Order.query()
  .groupBy("status")
  .having("count", ">", 10);
```

SQL:

```sql
HAVING count > 10;
```

Used with grouped results.

---

# Joining Tables

Joins combine data from multiple tables.

Example:

```text
Users
  │
  └── Orders
```

The Query Builder supports SQL joins while maintaining readability.

---

# Inner Joins

Example:

```ts
await User.query()
  .join(
    "orders",
    "users.id",
    "orders.user_id"
  );
```

SQL:

```sql
INNER JOIN orders
ON users.id =
orders.user_id;
```

---

# Left Joins

Example:

```ts
await User.query()
  .leftJoin(
    "orders",
    "users.id",
    "orders.user_id"
  );
```

This includes records even when relationships do not exist.

---

# Searching

Example:

```ts
await User.query()
  .whereLike(
    "name",
    "%john%"
  );
```

SQL:

```sql
WHERE name
LIKE '%john%';
```

Useful for search functionality.

---

# Conditional Queries

Sometimes conditions are applied dynamically.

Example:

```ts
User.query()
  .when(
    search,
    (query) => {
      query.whereLike(
        "name",
        `%${search}%`
      );
    }
  );
```

Benefits:

```text
Cleaner Code

Reusable Queries

Conditional Logic
```

---

# Query Reuse

Queries can be composed and reused.

Example:

```ts
const query =
  User.query()
    .where(
      "active",
      true
    );
```

This improves maintainability.

---

# Query Scopes

Scopes encapsulate common query patterns.

Example:

```ts
User.active();
```

Instead of:

```ts
User.query()
  .where("active", true);
```

Scopes improve readability and consistency.

---

# Raw Expressions

Occasionally raw SQL is required.

Example:

```ts
User.query()
  .selectRaw(
    "COUNT(*) AS total"
  );
```

Raw expressions should be used carefully and only when necessary.

---

# Transactions

Queries can execute within transactions.

Example:

```ts
await db.transaction(
  async (trx) => {
    //
  }
);
```

Transactions ensure data consistency.

# Best Practices

## Retrieve Only Needed Columns

Prefer:

```ts
.select("id", "name")
```

over:

```ts
SELECT *
```

whenever possible.

---

## Use Pagination

Avoid loading thousands of records at once.

---

## Reuse Query Logic

Create scopes and reusable query methods.

---

## Prefer Query Builder Over Raw SQL

Use framework abstractions when possible.

---

## Index Frequently Queried Columns

Database indexes dramatically improve performance.

Raw SQL reduces maintainability and portability.

---
