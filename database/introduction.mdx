# Introduction

Almost every application needs a place to store information.

Whether you're building:

- A blog
- An e-commerce platform
- A SaaS product
- A social network
- An internal business system

your application will eventually need to persist data beyond a single request.

The Database layer is responsible for storing, retrieving, updating, and deleting that data.

Bejibun provides a structured database system that helps developers interact with databases efficiently while maintaining clean, maintainable application code.

---

# What Is A Database?

A database is a system used to store and organize information.

Unlike temporary application memory, database data persists even after the application stops running.

Example:

```text
Application Stops
        │
        ▼
Database Data Remains
```

This allows information to survive:

- Server restarts
- Deployments
- Application crashes
- Infrastructure changes

---

# Why Databases Matter

Consider a user registration endpoint.

Request:

```json
{
  "name": "John Doe",
  "email": "john@example.com"
}
```

Without a database:

```text
Data Exists Only In Memory
```

After restarting the application:

```text
User Data Lost
```

With a database:

```text
Data Persisted Permanently
```

The user can return later and still access their account.

# Why Not Use Raw SQL Everywhere?

Direct SQL works well for small examples.

Example:

```sql
SELECT *
FROM users;
```

However, as applications grow:

```text
Queries Multiply

Logic Becomes Repetitive

Maintenance Becomes Difficult
```

Problems often include:

```text
Duplicate Queries

Inconsistent Logic

Security Risks

Poor Maintainability
```

Framework abstractions help solve these challenges.

# Database Concepts

Before diving deeper, it's important to understand several core concepts.

---

## Tables

A table stores related data.

Example:

```text
users
```

| id | name | email |
|----|----|----|
| 1 | John | john@example.com |
| 2 | Jane | jane@example.com |

Tables are the foundation of relational databases.

---

## Rows

Each entry inside a table is called a row.

Example:

| id | name |
|----|----|
| 1 | John |

This row represents a single user record.

---

## Columns

Columns define the structure of stored data.

Example:

```text
id

name

email

created_at
```

Every row follows the same column structure.

---

## Primary Keys

A primary key uniquely identifies a record.

Example:

```text
id = 1
```

Benefits:

```text
Uniqueness

Fast Lookup

Relationship Support
```

Most tables contain a primary key.

---

## Relationships

Tables can be connected together.

Example:

```text
User
 │
 └── Posts
```

One user may own many posts.

Relationships allow applications to model real-world data.


```text

```

# Transactions

Some operations must succeed together.

Example:

```text
Create Order
      │
      ▼
Create Payment
      │
      ▼
Create Invoice
```

If any step fails:

```text
Rollback Everything
```

Transactions help maintain data integrity.
