---
title: "File Uploads"
description: "Receive uploaded files via parse() and persist them with the Storage facade."
icon: "upload"
---

File uploads flow through the same `super.parse(request)` you use for any other input. Uploaded files arrive as native `File` objects, which you then persist with the [`Storage`](/) facade.

## Receiving an upload

For a `multipart/form-data` request, `parse()` includes uploaded files alongside regular fields:

```ts app/controllers/UploadController.ts
import BaseController from "@bejibun/core/bases/BaseController";
import Storage from "@bejibun/core/facades/Storage";

export default class UploadController extends BaseController {
    public async store(request: Bun.BunRequest): Promise<Response> {
        const body = await super.parse(request);

        const file = body.avatar as File;       // the uploaded file
        const content = await file.arrayBuffer();

        await Storage.disk("public").put(`avatars/${file.name}`, content);

        return super.response
            .setMessage("Uploaded")
            .setData({ name: file.name, size: file.size })
            .send();
    }
}
```

## Storing files

The `Storage` facade writes to the disks configured in `config/disk.ts` (a `local` and a `public` disk ship by default):

<CodeGroup>
```ts Default disk
await Storage.put("uploads/report.pdf", content);
```

```ts Specific disk
await Storage.disk("public").put("avatars/photo.png", content);
```

```ts Ad-hoc disk
import App from "@bejibun/app";

await Storage.build({
    driver: "local",
    root: App.Path.storagePath("custom")
}).put("file.txt", content);
```
</CodeGroup>

The `public` disk is rooted at `storage/app/public` and exposes a public URL via `APP_URL/storage/public`.

## Validating uploads

Validate the rest of the form payload as usual before storing:

```ts
const body = await super.parse(request);
await super.validate(UploadValidator.store, body);

const file = body.avatar as File;
await Storage.disk("public").put(`avatars/${file.name}`, await file.arrayBuffer());
```

<Tip>
For cloud storage, configure the `s3` disk in `config/disk.ts` with your S3 credentials, then call `Storage.disk("s3").put(...)`. The local and S3 drivers share the same API.
</Tip>

<Warning>
Never trust the client-supplied filename directly. Sanitize or generate your own name (and validate the content type / size) before writing to disk.
</Warning>
