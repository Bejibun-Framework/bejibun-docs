# Installation

Getting started with Bejibun is straightforward.

This guide will walk you through the requirements, installation process, and verification steps needed to create a new Bejibun application.

---

# Requirements

Before installing Bejibun, ensure your development environment meets the minimum requirements.

## Bun

Bejibun is built specifically for Bun and requires a recent Bun release.

Check your current version:

```bash
bun --version
```

If Bun is not installed, follow the official installation instructions from:

```text
https://bun.sh
```

---

## Node.js (Optional)

Some development tools may still rely on the Node.js ecosystem.

While Bejibun runs on Bun, having a recent Node.js version available can be helpful when working with third-party tooling.

---

## Database

Depending on your application, you may need one of the supported database systems.

Common choices include:

```text
PostgreSQL
MySQL
SQLite
```

SQLite is recommended for local development and tutorials.

---

## Redis (Optional)

Redis is required only if your application uses features such as:

- Cache
- Queues
- Rate limiting
- Realtime features

For basic applications, Redis is optional.

---

# Creating a New Application

Create a new Bejibun project using the framework installer.

```bash
bun create bejibun my-app
```

Replace `my-app` with your preferred project name.

Example:

```bash
bun create bejibun blog-api
```

After installation:

```bash
cd blog-api
```

---

# Project Creation Process

The installer will generate a complete application structure.

Example:

```text
blog-api/
├── app/
├── bootstrap/
├── config/
├── database/
├── public/
├── routes/
├── storage/
├── .env
├── package.json
└── tsconfig.json
```

This structure provides everything needed to begin development.

---

# Install Dependencies

If dependencies are not automatically installed, run:

```bash
bun install
```

This installs all required framework packages and project dependencies.

---

# Environment Configuration

Create your environment file if one does not already exist.

```bash
cp .env.example .env
```

Open the file and configure your application settings.

Example:

```env
APP_NAME=Bejibun
APP_ENV=development
APP_DEBUG=true
APP_URL=http://localhost:3000
```

Additional configuration options are covered in the Environment Variables guide.

---

# Generate Application Key

If your version of Bejibun requires an application key, generate one using:

```bash
bun ace key:generate
```

This key is used by framework features that require cryptographic security.

---

# Configure the Database

Update your database configuration inside the `.env` file.

### SQLite

```env
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite
```

Create the database file:

```bash
touch database/database.sqlite
```

---

### PostgreSQL

```env
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=bejibun
DB_USERNAME=postgres
DB_PASSWORD=password
```

---

### MySQL

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=bejibun
DB_USERNAME=root
DB_PASSWORD=password
```

Choose the configuration that matches your environment.

---

# Run Database Migrations

Create the database schema:

```bash
bun ace migrate
```

Expected output:

```text
Migrating...
Migration completed successfully.
```

Your application database is now ready.

---

# Seed the Database (Optional)

If your project includes seeders:

```bash
bun ace db:seed
```

This populates the database with initial data.

---

# Start the Development Server

Launch the application:

```bash
bun dev
```

Or:

```bash
bun start
```

depending on your project configuration.

Expected output:

```text
Server running on http://localhost:3000
```

---

# Verify Installation

Open your browser and visit:

```text
http://localhost:3000
```

If installation was successful, you should see the default Bejibun response or welcome page.

You can also test using:

```bash
curl http://localhost:3000
```

Example response:

```json
{
  "message": "Welcome to Bejibun"
}
```

# Common Commands

During development, you'll frequently use the following commands.

## Start Development Server

```bash
bun dev
```

---

## Run Migrations

```bash
bun ace migrate
```

---

## Roll Back Migrations

```bash
bun ace migrate:rollback
```

---

## Seed Database

```bash
bun ace db:seed
```

---

## Generate Controller

```bash
bun ace make:controller UserController
```

---

## Generate Model

```bash
bun ace make:model User
```

---

## Generate Migration

```bash
bun ace make:migration create_users_table
```

---

# Verify Framework Installation

You can confirm that Bejibun is properly installed by checking available commands.

```bash
bun ace list
```

Example output:

```text
make:controller
make:model
make:migration
migrate
db:seed
...
```

If commands appear correctly, the framework has been installed successfully.

# Troubleshooting

## Bun Not Found

Error:

```text
bun: command not found
```

Solution:

Install Bun and ensure it is available in your system path.

---

## Migration Failures

Check:

```env
DB_HOST
DB_PORT
DB_DATABASE
DB_USERNAME
DB_PASSWORD
```

and verify that the database server is running.

---

## Port Already In Use

Error:

```text
Address already in use
```

Solution:

Change the application port or stop the process using the port.

---

## Missing Environment Variables

Ensure:

```env
APP_ENV
APP_URL
DB_CONNECTION
```

and any required credentials are correctly configured.
