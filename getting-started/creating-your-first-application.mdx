# Creating Your First Application

In this guide, you'll build a simple API using Bejibun and become familiar with the framework's core concepts.

By the end, you'll understand how routes, controllers, validation, and models work together to handle requests.

---

# What We'll Build

We'll create a simple User API that allows clients to:

```text
GET    /users
POST   /users
GET    /users/:id
DELETE /users/:id
```

The final application will demonstrate:

- Routing
- Controllers
- Validation
- Database interaction
- JSON responses

---

# Create a New Project

If you haven't already created a project:

```bash
bun create bejibun my-app
```

Navigate into the project:

```bash
cd my-app
```

Install dependencies:

```bash
bun install
```

Start the development server:

```bash
bun dev
```

---

# Understanding the Request Flow

A typical request moves through the application like this:

```text
Request
   │
   ▼
Route
   │
   ▼
Controller
   │
   ▼
Validation
   │
   ▼
Model
   │
   ▼
Database
   │
   ▼
Response
```

Most Bejibun applications follow this pattern.

# Create Your First Route

Open:

```text
routes/api/index.ts
```

Add a route:

```ts
Router.get("/", () => {
  return {
    message: "Hello Bejibun"
  };
});
```

Start the server and visit:

```text
http://localhost:3000
```

Response:

```json
{
  "message": "Hello Bejibun"
}
```

Congratulations—you've created your first Bejibun route.

# Create a Controller

As applications grow, placing all logic inside routes becomes difficult to maintain.

Controllers help organize request handling.

Generate a controller:

```bash
bun ace make:controller UserController
```

Example:

```text
app/
└── controllers/
    └── UserController.ts
```

Open the controller:

```ts
export default class UserController {
  async index() {
    return [];
  }
}
```

---

# Register the Controller Route

Update your route:

```ts
import UserController from "#controllers/UserController";

Router.get(
  "/users",
  [UserController, "index"]
);
```

When the route is called, Bejibun executes the controller method.

---

# Create a Model

Models represent data stored in the database.

Generate a model:

```bash
bun ace make:model User
```

Example:

```text
app/
└── models/
    └── User.ts
```

Basic model:

```ts
export default class User {
  //
}
```

Models provide a structured way to interact with database records.

---

# Create a Migration

Generate a migration:

```bash
bun ace make:migration create_users_table
```

Example structure:

```text
database/
└── migrations/
```

Migration example:

```ts
export default class CreateUsersTable {
  async up() {
    //
  }

  async down() {
    //
  }
}
```

Migrations define database schema changes.

---

# Run Migrations

Apply migrations:

```bash
bun ace migrate
```

This creates the required database tables.

---

# Return Database Records

Update the controller:

```ts
import User from "#models/User";

export default class UserController {
  async index() {
    return await User.query();
  }
}
```

Now:

```http
GET /users
```

returns all users stored in the database.

---

# Create a Validator

Validation protects your application from invalid input.

Generate a validator:

```bash
bun ace make:validator CreateUserValidator
```

Example:

```text
app/
└── validators/
    └── CreateUserValidator.ts
```

Basic validator:

```ts
import { validator } from "@bejibun/validator";

export default class CreateUserValidator {
  schema = {
    name: validator.string(),
    email: validator.email()
  };
}
```

Validation ensures requests contain the expected data.

---

# Create Users

Add a new controller method:

```ts
export default class UserController {
  async store({ request }) {
    const data =
      await request.validate(
        CreateUserValidator
      );

    return await User.create(data);
  }
}
```

Register the route:

```ts
Router.post(
  "/users",
  [UserController, "store"]
);
```

Request:

```http
POST /users
```

Body:

```json
{
  "name": "John Doe",
  "email": "john@example.com"
}
```

Response:

```json
{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com"
}
```

---

# Retrieve a Single User

Add another method:

```ts
async show({ params }) {
  return await User
    .query()
    .findById(params.id);
}
```

Register route:

```ts
Router.get(
  "/users/:id",
  [UserController, "show"]
);
```

Request:

```http
GET /users/1
```

Response:

```json
{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com"
}
```

---

# Delete a User

Add:

```ts
async destroy({ params }) {
  await User
    .query()
    .deleteById(params.id);

  return {
    success: true
  };
}
```

Register route:

```ts
Router.delete(
  "/users/:id",
  [UserController, "destroy"]
);
```

Request:

```http
DELETE /users/1
```

Response:

```json
{
  "success": true
}
```

# Organizing Your Application

After adding routes, controllers, models, and validators, your project might look like:

```text
app/
├── controllers/
│   └── UserController.ts
│
├── models/
│   └── User.ts
│
├── validators/
│   └── CreateUserValidator.ts
│
└── exceptions/

database/
└── migrations/

routes/
└── api/
```

Each component has a clear responsibility.

# Using Middleware

Middleware can protect routes before they reach controllers.

Example:

```ts
Router.get(
  "/users",
  [UserController, "index"]
).middleware(["auth"]);
```

Common middleware includes:

- Authentication
- Authorization
- Logging
- Rate limiting

---

# Returning Responses

Routes and controllers can return JSON directly.

Example:

```ts
return {
  message: "Success"
};
```

Response:

```json
{
  "message": "Success"
}
```

Bejibun automatically serializes objects into JSON responses.

---

# Error Handling

Throw exceptions when operations fail.

Example:

```ts
throw new Error(
  "User not found"
);
```

The framework's exception handler will:

```text
Capture Error
      │
      ▼
Log Error
      │
      ▼
Generate Response
```

This keeps error handling consistent throughout the application.
