# MVC Pattern

Bejibun follows the **Model-View-Controller (MVC)** architectural pattern to help developers build maintainable, scalable, and organized applications.

MVC separates application responsibilities into three distinct layers:

* **Models** handle data and business logic.
* **Views** represent the data returned to clients.
* **Controllers** process incoming requests and coordinate application flow.

This separation makes code easier to understand, test, and maintain as your application grows.

## Why MVC?

Without a clear architecture, application logic often becomes scattered across routes, services, and database queries, making projects difficult to maintain.

MVC provides:

* Clear separation of concerns
* Better code organization
* Improved maintainability
* Easier testing
* Reusable business logic
* Consistent project structure

A typical request flows through the application as follows:

1. A client sends an HTTP request.
2. A route matches the request.
3. The route invokes a controller action.
4. The controller performs application logic.
5. Models interact with the database.
6. A response is returned to the client.

## Models

Models represent application data and provide an interface for interacting with the database.

In Bejibun, models are responsible for:

* Querying records
* Creating and updating data
* Defining relationships
* Encapsulating data-related behavior

### Example

```ts
import { Model } from "@bejibun/core";

export class User extends Model {
  protected table = "users";
}
```

### Responsibilities

Good model responsibilities include:

* Data persistence
* Query scopes
* Relationships
* Data transformations

Avoid placing HTTP-specific logic inside models.

```ts
// вќЊ Avoid
return response.json(user);
```

Models should remain independent of the transport layer.

## Controllers

Controllers receive incoming requests and return responses.

They act as the bridge between HTTP requests and your application's business logic.

### Example

```ts
import { Request, Response } from "@bejibun/core";
import { User } from "@/models/User";

export class UserController {
  async index(req: Request, res: Response) {
    const users = await User.all();

    return res.json(users);
  }
}
```

### Responsibilities

Controllers should:

* Validate incoming requests
* Call services or models
* Coordinate application flow
* Return responses

Controllers should remain thin whenever possible.

```ts
// Good
const user = await userService.create(data);

// Avoid
// Hundreds of lines of business logic directly
// inside controller methods.
```

## Views

Traditional MVC frameworks often use server-rendered templates.

Bejibun is API-first, so a "view" commonly refers to the data representation returned to clients.

Examples include:

* JSON responses
* API Resources
* Serialized objects
* Server-side rendered templates (optional)

### JSON Response

```ts
return res.json({
  id: user.id,
  name: user.name,
});
```

### API Resource

```ts
return UserResource.make(user);
```

API Resources provide a dedicated layer for transforming data before it reaches clients.

## Keeping Controllers Thin

As applications grow, controllers can become difficult to maintain.

Instead of placing business logic directly inside controllers, move complex operations into dedicated services.

### Not Recommended

```ts
export class OrderController {
  async store(req, res) {
    // Validation
    // Payment processing
    // Inventory updates
    // Notifications
    // Logging

    return res.json({});
  }
}
```

### Recommended

```ts
export class OrderController {
  constructor(
    private orderService: OrderService
  ) {}

  async store(req, res) {
    const order = await this.orderService.create(
      req.body()
    );

    return res.json(order);
  }
}
```

This approach improves:

* Readability
* Reusability
* Testability
* Separation of concerns

## MVC and the Service Container

Bejibun integrates MVC with the Service Container and Dependency Injection system.

Controllers can receive dependencies automatically from the container.

```ts
export class UserController {
  constructor(
    private userService: UserService
  ) {}

  async index() {
    return this.userService.all();
  }
}
```

This allows controllers to focus on application flow while services handle business logic.

## Summary

The MVC pattern is the foundation of application structure in Bejibun.

By separating responsibilities between models, controllers, and views, you can build applications that are easier to understand, test, and scale.

As you continue through the Architecture section, you'll learn how MVC integrates with Bejibun's Service Container, Dependency Injection system, and Service Providers.
